  ▎
  ▎   A. Planner + prompt tuning from the Phase 2 live-run caveats (still open). The
  ▎      Phase 2 finale flagged three concrete issues: (1) the planner over-parallelises
  ▎      modules that share output files, causing merge conflicts when concurrent builders
  ▎      write the same file — planner prompt needs to model file ownership and emit
  ▎      dependsOn on any task that writes to a file a prior task also writes; (2) the

  ▎      pool-level clamp so `agent: 'builder'` can't end up below `reasoning`; (3) the
  ▎      default max-turns of 20 is tight for larger builder tasks — either raise globally
  ▎      via ClaudeCliProviderOptions.maxTurns or let the planner suggest per-task turn
  ▎      budgets. Cheapest direct win for build quality. Budget $5-15 to re-run
  ▎      `factory build example --autonomy autonomous --concurrency 2` after each tuning
  ▎      pass until all 14 tasks succeed and the verify gate is green. Record spend + task
