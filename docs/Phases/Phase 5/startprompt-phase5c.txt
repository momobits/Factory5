Step 1 — Fix I002 (assessor env), minimum-viable tier.

Code changes live in `packages/assessor/src/runners/pytest.ts` and its
test file. The detailed design is in I002 under "Resolution → Minimum
viable (one session)"; do not wander beyond tier 1.

  a. `pickPython(opts)` gains venv preference + requires-python selection.
     Use `smol-toml` (already in the workspace via brain.config) to parse
     pyproject.toml when present. Priority order:
       1. caller-provided `opts.pythonBin`
       2. `<projectPath>/.venv/Scripts/python.exe` (Windows) or
          `<projectPath>/.venv/bin/python` (Unix) — whichever exists
       3. if `pyproject.toml` has `requires-python >= 3.X`, try `py -3.X`
          (Windows) / `python3.X` (Unix) in descending order from the most
          recent available
       4. fallback to bare `python` / `python3`
     Log at `info` with `{ chosen, reason }` so the assessor's decision is
     traceable.

  b. Before invoking pytest, run `<python> -m pip install -e .[test]`
     against the chosen interpreter. If pyproject has no `[project.optional-dependencies.test]`, retry with plain `-e .`. Capture
     stdout/stderr; surface failures as `provisioning.installOk: false`
     with the last 40 lines of stderr in `installSummary`.

  c. Extend `AssessResult` (in `packages/assessor/src/types.ts`) with
     `provisioning?: { pythonPath: string; pythonVersion: string;
     installOk: boolean; installSummary?: string }`. Thread it through
     `runPytest` → `assess` → the log line `assess: complete`.

  d. New tests in `pytest.test.ts`:
     - venv detection picks `.venv/Scripts/python.exe` when present
     - requires-python >= 3.11 picks py -3.11 when available, falls back
       to py -3.10 (or whatever) when it isn't, and logs the demotion at warn
     - install step runs before pytest
     - install failure surfaces on `provisioning.installOk = false` and
       the gate result is `{ build: false }` rather than `{ build:
       test-exit-5 }`

  e. `docs/decisions/0017-assessor-project-env-provisioning.md` — new
     ADR documenting the tier-1 decision, why tier-2 (`.factory/assessor-env/`)
     and tier-3 (pluggable runtimes) are deferred. Reference I002. Use
     ADR 0016's shape (Context / Decision / Consequences / Alternatives).

After each concrete edit: `pnpm build`, `pnpm --filter @factory5/assessor test`, `pnpm lint`, `pnpm format:check`. No live run yet.
