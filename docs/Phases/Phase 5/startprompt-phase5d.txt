Step 1 — Fix I003 (scaffolder project hygiene), prompt-tuning tier.

All code changes are prompt text. Details in I003 under
"Resolution → Suggested direction (tier 1, prompt-only)"; stay inside
that scope.

  a. `prompts/agents/scaffolder.md`:
     - Add an explicit "repo-level hygiene files" section to the scaffolder's
       default outputs: `README.md` (≥30 non-empty lines, substantive —
       overview + install + usage + test-how-to + license reference),
       `LICENSE` (default MIT with the current year and a sensible
       placeholder author; the spec can override), and a runtime-aware
       `.gitignore`.
     - For Python projects: `.gitignore` must cover `__pycache__/`,
       `*.pyc`, `.pytest_cache/`, `.coverage`, `htmlcov/`, `*.egg-info/`,

       non-empty lines. Stub files fail the gate.

  b. `prompts/agents/architect.md`:
     - Add a rule in the wiki-scope guidance that `overview.md` must
       cover the project's repo-level hygiene intent: what the README

       which runtime's `.gitignore` patterns are needed. This makes the
       scaffolder's job deterministic.

  c. `packages/brain/src/architect.ts` — read the inline user prompt. If
     it duplicates guidance from architect.md, update it in parallel so

  d. `packages/brain/src/agents/registry.ts` — check whether `scaffolder`

Step 2 — Author a parallel-admitting spec.

I001 validation requires a project where the architect's natural design
puts two modules at the same DAG level, sharing only the scaffolder. The
current `example` spec is a weather CLI where every module transitively
reads `models` or `api`, so it can never parallelise.

  a. Create `templates/parallel-example/CLAUDE.md`.
     - Small (~25-40 lines), in the same shape as `templates/example/CLAUDE.md`.
     - Pick a project that genuinely has independent modules. For example:
       a CLI with two unrelated utilities — say a JSON prettifier and an
       ASCII-art text renderer — plus a tiny dispatcher. The utilities share
       zero imports with each other; the dispatcher imports both.
     - Expected DAG after planner runs: scaffolder → {prettifier builder,
       art builder} (both `dependsOn: [0]`, no edge between them) →
       dispatcher builder → verifier. Two parallel builders at level 1.
     - Don't include any source files in the template — the factory builds
       it from scratch. Only `CLAUDE.md`.

  b. Keep the spec tight. Tech stack: Python 3.11+, stdlib-only if
     possible (to keep pip install fast). Each utility has its own tests;
     the dispatcher has tests that exercise both.

  c. No README/LICENSE in the template — the scaffolder should produce
     them (that's Step 1's fix).

After: `pnpm build`, `pnpm test`, `pnpm lint`, `pnpm format:check`. No
TS changes expected beyond possibly the architect/scaffolder tweaks in
Step 1.
